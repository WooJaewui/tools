 # 웹 프로젝트로 변경하기.

설명.
pom.xml에서 <packaging>jar를 war로 바꾸면 웹 프로젝트로 변경된다.

사용.
<packaging>war</packaging>
project -> maven -> Update Project.
src -> main -> webapp -> WEB-INF 폴더 생성 -> web.xml 생성.

추가설명.
Update Project를 하면 project 구조가 웹 프로젝트로 변경된다.



 # 서블릿/JSP 라이브러리 설정하기.

build path 설정.
tomcat 폴더를 추가.
tomcat/lib 폴더에 jsp.lib와 servlet.lib를 포함하고 있다.

pom.xml <dependency> 설명.
Maven은 웹 어떠한 저장소에 라이브러리와 모듈 등을 저장해놓는다. ( Central Repository )
내가 dependency를 통해 어떤 자료가 필요하다고 설정하면, 
내 컴퓨터 Maven 저장소( Local Repository )에 웹 저장소에 있는 데이터를 가져와서 사용할 수 있도록 한다.
내 컴퓨터 Maven에 저장된 라이브러리는 다른 프로젝트에서도 사용가능하다.

Central Repository 검색.
google에 Maven Repository 입력 -> 원하는 lib 검색 -> 버전 선택 -> Maven 클릭 코드 복사 -> pom.xml 입력.

검색팁.
JSP 라이브러리가 필요한데 버전을 잘 모르는 경우. 
Tomcat JSP를 검색하면 Tomcat 버전으로 JSP를 설정할 수 있다.

Maven 라이브러리 확인.
C드라이브 -> 사용자 -> user이름 -> .m2 -> repository 안에 저장됨.
C드라이브 -> 사용자 -> user이름 -> .m2 -> repository -> org -> apache -> tomcat 라이브러리.



 # 라이브러리 오류 문제.

jar 파일 열어 보는 방법.
파일이름.jar를 파일이름.zip 압축파일로 바꿔서 확인할 수 있다.

라이브러리 오류.
라이브러리의 일부 파일이 손상되거나 했을 때 문제가 발생한다.

라이브러리 오류 확인.
라이브러리의 세부 내용을 확인해 봤을때, 클래스 등 안에 내용이 확인이 안되면 손상된 것이다.

오류 해결하기.
이클립스 종료 -> C드라이브 -> 사용자 -> user이름 -> .m2 -> repository 폴더 안에 있는 파일 삭제하기.
